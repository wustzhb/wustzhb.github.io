title: hexo博客搭建填坑

hexo博客搭建填坑

参考视频（感谢WEB群的同学分享）：https://www.bilibili.com/video/BV1Yb411a7ty?p=1&share_medium=android&share_plat=android&share_source=QQ&share_tag=s_i&timestamp=1618232003&unique_k=3r1JET

1.提前安装好Git，否则无法使用hexo类命令

2.使用hexo命令要在git bash中使用,在执行命令前先进入到你想要搭建的文件目录

3.gitbash here如果提示无法找到应用，那么应该是你改变了默认的安装目录，则需要在注册表的dictionary中的2个git注册表command重新编辑所在的安装目录（如果不改安装目录默认安装则一点事没有），但我在改完后仍然会报错，索性直接在git中用bash然后手敲进博客的目录。。

4.如果在hexo init过程中报错，那么有如下原因:
（1）github被墙导致报错timeout，所以要修改host文件（可以用站长工具DNS下github.com，选TTL最小的DNS）

​ (2)文件夹不是空的导致报错（一堆at开头的），把当前文件夹清空了就行

​ (3)还是更改默认安装目录的锅，导致ssl证书找不到，用git config –system http.sslcainfo “正确路径”在bash下输入就OK

​ (4)如果npm或者node之类的命令输入后提示不是命令，那么一定是环境变量没配置好

​ (5)init若直接使用hexo类命令可能会报错：

ERROR Cannot find module ‘hexo’ from ‘D:\blog_hexo’
ERROR Local hexo loading failed in D:\blog_hexo
ERROR Try running: ‘rm -rf node_modules && npm install –force’

csdn上说导致这个原因为在自己的博客文件夹中,.gitignore文件中添加了node_modules/,导致更新的时候,这个文件夹被忽略,没有被更新上去。

解决方法：执行 npm instal 命令
